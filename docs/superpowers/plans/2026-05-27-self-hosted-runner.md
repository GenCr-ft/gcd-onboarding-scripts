# Self-Hosted GitHub Actions Runner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate GitHub-hosted runner billing by registering a persistent self-hosted runner on the WSL2 machine at org level and migrating all active code repo CI workflows to target it.

**Architecture:** One org-level runner registered under `GenCr-ft`, run as a systemd service (systemd already enabled in WSL2 — confirmed by `/etc/wsl.conf`). All code repo `ci.yml` files change `runs-on: ubuntu-latest` → `runs-on: [self-hosted, linux, x64]`. The `ssot-compliance.yml` reusable workflow (called at tag `@v1.2.5`) stays on `ubuntu-latest` for now — it is `continue-on-error: true` and already fails due to missing `CROSS_REPO_PAT`; a separate issue covers migrating it.

**Tech Stack:** GitHub Actions Runner v2.334.0, systemd, bash, gh CLI

**Pre-confirmed on this machine:** Node.js v25.9.0, Python 3.12.3, Rust 1.95.0, Docker 29.1.3, wasm-pack 0.13.1, Godot 4.5 binary at `/home/lgan/hxgn/dev/claude/exp/Godot_v4.5-stable_linux.x86_64`

**Out of scope (separate issue):**
- `gcd-shared-actions` reusable workflow `runs-on` migration (touches all 20+ SSoT compliance callers — new semver tag required)
- `gcd-onboarding-scripts` macOS matrix job (macOS runner would need a second machine)
- `gcl-ui-components`, `gcl-voxel-engine` (stub/on-ice repos)

- [ ] **Step 1: Write the service unit file**

```bash
sudo tee /etc/systemd/system/github-runner.service > /dev/null <<'EOF'
[Unit]
Description=GitHub Actions Runner — GenCr-ft org (lgan-wsl2)
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=lgan
WorkingDirectory=/home/lgan/actions-runner
ExecStart=/home/lgan/actions-runner/run.sh
Restart=on-failure
RestartSec=5
KillMode=process
KillSignal=SIGTERM
TimeoutStopSec=5min

# Forward stdout/stderr to journald
StandardOutput=journal
StandardError=journal
SyslogIdentifier=github-runner

# Toolchain paths (WSL2 PATH is not inherited from shell profile)
Environment="PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/home/lgan/.cargo/bin:/home/lgan/.local/bin"
Environment="HOME=/home/lgan"
Environment="GODOT_BIN=/home/lgan/hxgn/dev/claude/exp/Godot_v4.5-stable_linux.x86_64"

[Install]
WantedBy=multi-user.target
EOF
```

## Task 5: Fix Godot 4.5 Availability for Client CI

**Context:** `/usr/local/bin/godot` currently points to Godot 4.2. The client CI downloads and installs Godot 4.5 on each run — wasteful on a self-hosted runner where the binary already exists. We update the install step to skip the download if Godot 4.5 is already present.

**Files:**
- Modify: `gcp-aethel-client/.github/workflows/ci.yml:19-24` (Install Godot step)

- [ ] **Step 1: Verify the 4.5 binary is executable**

```bash
/home/lgan/hxgn/dev/claude/exp/Godot_v4.5-stable_linux.x86_64 --version 2>/dev/null
```

Expected: `4.5.stable.official.<hash>`

- [ ] **Step 2: Replace the Install Godot step with a version-aware variant**

In `gcp-aethel-client/.github/workflows/ci.yml`, replace the current `Install Godot 4.5 (headless)` step:

```yaml
      - name: Install Godot 4.5 (headless)
        run: |
          wget -q https://github.com/godotengine/godot/releases/download/4.5-stable/Godot_v4.5-stable_linux.x86_64.zip
          unzip -q Godot_v4.5-stable_linux.x86_64.zip
          chmod +x Godot_v4.5-stable_linux.x86_64
          sudo mv Godot_v4.5-stable_linux.x86_64 /usr/local/bin/godot
```

Replace with:

```yaml
      - name: Install Godot 4.5 (headless, skip if already present)
        run: |
          TARGET="${GODOT_BIN:-/usr/local/bin/godot}"
          if "$TARGET" --version 2>/dev/null | grep -q "^4\.5"; then
            echo "Godot 4.5 already available at $TARGET — skipping download."
            sudo ln -sf "$TARGET" /usr/local/bin/godot 2>/dev/null || true
          else
            wget -q https://github.com/godotengine/godot/releases/download/4.5-stable/Godot_v4.5-stable_linux.x86_64.zip
            unzip -q Godot_v4.5-stable_linux.x86_64.zip
            chmod +x Godot_v4.5-stable_linux.x86_64
            sudo mv Godot_v4.5-stable_linux.x86_64 /usr/local/bin/godot
          fi
```

- [ ] **Step 3: Verify the edit is correct**

```bash
grep -A12 "Install Godot" /home/lgan/hxgn/dev/claude/exp/gcp-aethel-client/.github/workflows/ci.yml
```

Expected: the new version-check variant is present.
